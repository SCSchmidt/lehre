Die Syntax dafr ist immer gleich:

Die neue Funktion heit myfunction, sie wird auf eine Variable x angewandt und was mit x passiert, wird in den geschweiften Klammern definiert. x ist dabei der Platzhalter fr das Objekt, dass der Nutzer dann in die Funktion hineingeben wird.

Stellen wir uns vor, ich mchte eine Funktion fr die Berechnung des doppelten Mittelwertes von Vektoren. Die knnte z. B. so aussehen:

  2*sum(x)/length(x)
  }

Ich rechne zwei mal die Summe von x (d.h. x muss ein Vektor sein) und teile dies durch die Lnge des Vektors (Anzahl der Eintrge). Ich nenne die Funktion zwei_m.

zwei_m kann ich jetzt anwenden:

Tja. Doof, ne. Das Problem kennen wir ja schon. Im Datensatz sind NA-Werte, aber die Funktion hat noch keine Mglichkeit damit umzugehen.

Erweitern wir sie also!

zwei_m <- function(x){ 
  x <- na.omit(x)  # entferne aus x alle NA und berschreibe x damit ("berschreibung" passiert nur innerhalb der Formel)
  2*sum(x)/length(x)
  }

Testen wir es also:

## [1] 8403.509

Yay! In der Funktion habe ich definiert, dass das, was ich eingebe, erst durch die Funktion na.omit() geschickt wird und sich dabei selbst berschreibt. Damit wird dann weitergerechnet und - taadaaaaa - es gibt kein Problem mehr!

Schleifen

Grundstzlich lassen sich drei Formen von Schleifen unterscheiden:

  • die for-Schleife

  • die while-Schleife

  • die repeat (oder do-while) -Schleife

In die muss man sich jeweils etwas eindenken, aber so sehr komplex sind sie auch wieder nicht.

Die for-Schleife

Sinn und Zweck der for-Schleife ist es, einen Code x mal auszufhren. Die Variable x ist hierbei eine ganze Zahl (integer) oder jeder Wert innerhalb eines Vektors. Das kann zum Beispiel so aussehen:

x <- 5
for(i in 0:x) {
   print(i);
}
## [1] 0
## [1] 1
## [1] 2
## [1] 3
## [1] 4
## [1] 5

for i in ... ist dabei der Bereich, der sagt: fr jedes Element i in (oben dem Bereich 0 bis x, also 5) mache bitte das folgende { in geschweiften Klammern, hier: einfach nur print, d.h. drucke aus}.

Stelle ich mir vor, ich habe einen Vektor mit Worten darin, kann ich dies auch so machen:

x <- c("Mein", "Pinguin", "ist", "niedlich")
for(i in x) {
   print(i);
}
## [1] "Mein"
## [1] "Pinguin"
## [1] "ist"
## [1] "niedlich"

Die while-Schleife

Die while-Schleife dienst grundstzlich dazu, einen Code so lange auszufhren, wie eine bestimmte Bedingung erfllt ist. Das kann z. B. so aussehen:

i <- 0 

while (i < 5) {
   i <- i+1;
   print(i);
}
## [1] 1
## [1] 2
## [1] 3
## [1] 4
## [1] 5

Was passiert?

Solange i kleiner als 5 ist, wird: i + 1 gerechnet und damit i berschrieben und gedruckt.

Hier muss ich also immer eine Vernderung in der Schleife haben, die dazu fhrt, dass die while-Bedingung irgendwann erreicht ist, sonst luft sie unendlich weiter

Wie man merkt, wird dabei die Entscheidung, ob es weitergeht, immer erst oben in der Funktion getroffen. D.h. whrend die Funktion im Kreislauf luft, wird bis zum Ende alles ausgefhrt, wenn oben der while-Befehl noch nicht erfllt war. Deshalb wird i hier auch bis 5 gedruckt, denn im Kopf steht solange i kleiner ist als 5 addiere eins hinzu und druck es aus. Man nennt das Kopf-gesteuert.

Unter bestimmten Umstnden kann es sinnvoll sein, am Schleifenfu die Bedingung zu prfen, unter der die Schleife ausgefhrt wird. Diese Anforderung erfllt die

repeat-Schleife (do while-Schleife)

Die Syntax lautet:

i <- 0
repeat {
   i = i+1;
   if(i == 5) break;
   print(i)
}
## [1] 1
## [1] 2
## [1] 3
## [1] 4

Also, wiederhole (repeat) das addieren von 1 und drucken von i, aber wenn i == 5, dann unterbrich den Vorgang (break).

Damit haben wir eigentlich schon die nchste Mglichkeit der Steuerung von automatischen Prozessen kennengelernt: Die Bedingung.

Bedingungen

Normalerweise wird ein R-Code Zeile fr Zeile, von oben nach unten, ausgefhrt. Manchmal mchte man aber eine Zeile - oder einen ganzen Block von Zeilen - nur unter einer bestimmten Bedingung durchfhren. Dazu bietet sich in R die If-Anweisung an.

Dabei gilt immer if / falls (die Bedingung in der Runden Klammer wahr ist) {dann mach das in der geschweiften Klammer}:

d <- 100
  if (d >= 90) {
    print(paste("Sehr gut.", d , "ist grer als 90"))
    }
## [1] "Sehr gut. 100 ist grer als 90"

Also, wenn d grer 90 ist, dann drucke diesen zusammengefgten Satz (Funktion paste setzt einen string aus mehreren Teilen zusammen) sehr gut, der Wert d, ist grer als 90.

Ihr knnt ja mal ausprobieren, das d auf 85 zu setzen. Was passiert dann?

Richtig. Gar nichts.

Sehr hufig mchte man aber, dass dann etwas anderes bestimmtes passiert.

Dafr gibt es else if

  punkte <- 85
  if (punkte >= 90) {
    print(paste("Sehr gut.", d , "ist grer als 90"))
    } else if (punkte >= 80 & punkte < 90) {
    print("Gut");    
    } else {
      print(paste("Nicht gengend.", d , "ist zu klein"));

## [1] "Gut"

Also: Wenn (if) die Variable punkte grer als 90 ist, drucke Sehr gut

falls dem nicht so ist (else if) und die Variable Punkte liegt zwischen 80 und 90, drucke Gut

und falls keine dieser Bedingungen erfllt ist (else), drucke nicht gengend. der Wert ist zu klein.

Immer wird die Bedingung dabei in runde Klammern gesetzt und die Aktion in geschweifte.

Beispiel

Stellen wir uns vor, wir wollen shapiro-wilk fr das Gewicht aller drei Spezies Pinguine rechnen. Das sind drei Funktionen, kann ich da eine daraus machen? Und mir gemeinsam ausgeben lassen, welche Irrtumswahrscheinlichkeit bei jedem Test herauskommt?

Jetzt ist es die Kunst des Programmierens, sich genau zu berlegen, welche Schritte man dafr bentigt.

  1. Die Funktion braucht als Eingabe etwas, das den Datensatz in seine Gruppen splittet und die Funktion braucht als Eingabe die zu testende Variable.

  2. Ich brauche die einzelnen Werte fr die zu splittende Variable

  3. Ich muss die Gruppen anhand dieser Werte erstellen (subset)

  4. den Test pro Gruppe berechnen

  5. Ich muss die Teststatistik auslesen

  6. Ich muss dies in einer hbschen Tabelle ablegen.

  7. Die Tabelle mus ausgegeben werden.

# 1. Eingabeparameter definieren

sw_nachgruppe <- function(df, gruppe, variable){ 

# 2. die verschiedenen Werte des gruppiernden Merkmals abgreifen   
  
  level <- levels(as.factor(df[[gruppe]])) # levels zeigt, welche Werte in einem Faktor-Vektor vorkommen (mit as.factor sicherstellen, dass es nicht aus Versehen ein Character-Vektor ist)
  
  # (5.) Vorbereiten der Tabelle muss vor der Schleife passieren
  df_p <- data.frame(gruppe = character(length(level)),
                       pWert = double(length(level))) 
   # erstelle die Spalte gruppe als character-Spalte mit so vielen Zeilen, wie der Vektor "level" Eintrge hat (length)              

# Vobrereiten eines "Zhlers" fr 5.
   z <- 0
  
# Schleife fr Gruppenbildung und Berechnung:  
  for (i in level) { # fr jeden der Werte in level
    
#3. Gruppenbildung
  s <- subset(df, df[[gruppe]] == i) # filtere den Datensatz
  w <- s[[variable]] # weise die relevante Wertereihe einem Vektor zu
  
#4. Berechnung
  y <- shapiro.test(w) # berechne den Shapiro-Test auf die Variable
  
#5. + 6. Teststatistik auslesen und in Tabelle ablegen
     z <- z+1 # "hochzhlen", um die richtigen Zeilen des Ausgabe-Datensatzes anzuwhlen

    df_p$gruppe[z] <- i
    df_p$pWert[z] <- y$p.value
  }
  # Schleife zuende

 #7. Gesamtdatensatz ausgeben
  print(df_p)
}
